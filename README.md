## Reflection 3 — Use of GitHub Pages (STAR Method)

**Situation:**  
After completing my portfolio content, I needed to deploy it as a live, publicly accessible website so that potential employers and my lecturer could view it online. GitHub Pages was the required platform for this deployment.

**Task:**  
My task was to configure my GitHub repository to deploy my portfolio as a live website using GitHub Pages, ensuring the site was accessible at my personal GitHub Pages URL.

**Action:**  
I navigated to the Settings tab of my repository, located the Pages section under Code and Automation, and configured the source to deploy from the main branch. I waited for GitHub to build and publish the site, then verified it was live by visiting my GitHub Pages URL. I also ensured that my index.html and README.md files were correctly structured so the site rendered properly.

**Result:**  
My portfolio is now live and publicly accessible at [https://komaneOratilwe.github.io](https://komaneOratilwe.github.io). This experience taught me how to deploy a static website using GitHub Pages — a valuable and widely used skill in the industry. Having a live portfolio URL also gives me a professional online presence I can share directly with employers when applying for positions.
